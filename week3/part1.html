Exercise: Questions for chapter 7.

  1. Explain in your own words: What is the point of scales?

    2. Because input data points doesn’t necessarily corresponds to a physical dimension or size, like pixel, one can use a scale to scale the points up or down to suit the dimensions of the element that they are visualised in.

  2. Why are d3.min() and d3.max() useful? And what's the accessor function? When do you need that?

    3. d3.min() and d3.max() are used to make the scale dynamic. Say a new data point which is significantly different (in terms of size) from the rest is added, the min and max could be used to pick up the smallest and largest number in the data set and set the scales accordingly.
    An accessor function is to access a specific element from an object.
    It the context of scales it is needed when using d3.min() and d3.max() on multi dimensional arrays. The accessor function specifies which element in the multidimensional array should be used for comparison.

  3. What is Bostock's margin convention mentioned on page 125?

    4. Bostock's margin is a convention to create SVG elements with a margin, such that the margin can be ignored within the SVG element. See https://bl.ocks.org/mbostock/3019563

  4. Can you mention an example when you might need other scales than linear?

    5. If you have measurements that follows an exponential growth one could use a logarithmic scale.

  5. How do you convert strings to Date objects? When is it good to check that your conversion works? Do you have to be extra careful with Danish dates? Why?

    6. Create a function like var parseTime = d3.timeParse(“%m/%d/%y”)
    Before you start to use the converted data.
    One can parse the string differently like (“%d/%m/%y”). It is more important to understand the data and follow the convention for the domain you are working in.
