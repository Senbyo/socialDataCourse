Exercise: Questions for chapter 7.

  1. Explain in your own words: What is the point of scales?

    2. Because input data points doesn’t necessarily corresponds to a physical dimension or size, like pixel, one can use a scale to scale the points up or down to suit the dimensions of the element that they are visualised in.

  2. Why are d3.min() and d3.max() useful? And what's the accessor function? When do you need that?

    3. d3.min() and d3.max() are used to make the scale dynamic. Say a new data point which is significantly different (in terms of size) from the rest is added, the min and max could be used to pick up the smallest and largest number in the data set and set the scales accordingly.
    An accessor function is to access a specific element from an object.
    It the context of scales it is needed when using d3.min() and d3.max() on multi dimensional arrays. The accessor function specifies which element in the multidimensional array should be used for comparison.

  3. What is Bostock's margin convention mentioned on page 125?

    4. Bostock's margin is a convention to create SVG elements with a margin, such that the margin can be ignored within the SVG element. See https://bl.ocks.org/mbostock/3019563

  4. Can you mention an example when you might need other scales than linear?

    5. If you have measurements that follows an exponential growth one could use a logarithmic scale.

  5. How do you convert strings to Date objects? When is it good to check that your conversion works? Do you have to be extra careful with Danish dates? Why?

    6. Create a function like var parseTime = d3.timeParse(“%m/%d/%y”)
    Before you start to use the converted data.
    One can parse the string differently like (“%d/%m/%y”). It is more important to understand the data and follow the convention for the domain you are working in.

    Exercise: Questions for chapter 8.

  1. Explain in your own words: What is the point of axes?

    2. Axes are used to give the viewer of the data a sense of the scale of the data. Making such things at outliers visible and connecting the data point with each other as well as setting them in a common world.

  2. On page 136-7, Scott notes: "You might be wondering why this looks so different from our friendly scale functions". Answer in your own words.

    3. This looks different because the scale acts on the date, scaling it, and the axis do not. The axis are simply there to better our understanding of the data. They do not interact with the data. They do however interact with the elements that are being drawn on the screen, and as such need to be drawn last because they need to be drawn on top of everything else.

  3. What are SVG transformations?

    4. SVG transformations are transformation that can be done to the entire svg object. These include translation, as well as toration and scale.

  4. Why might D3 give you different axes-ticks than you ask for?

    5. When using tick(#) one is simply telling d3.js the prefered amount of ticks to use. However d3.js will overrule this if it causes awkward spacing in the ticks.

    Exercise: Questions for chapter 9.

  1. Explain in your own words, what is an ordinal scale? When do you need it?

    2. An ordinal scale is a scale for discrete categories that have a neutral order. Such as days of the week, or one ranging from, agree to disagree, or grades ranging from the lowest to the highest.

  2. If you were plotting data related to months (Jan, Feb, ... , Dec), would you use time-based scales or ordinal scales? Justify your answer.

    3. Using a time-based scale requres the use of JavaScripts time object. And as such I would only use the time-based scale if it made sense to convert the categories to Date objects. Typically if I just wanted to group based on day or month and the yScale was not time sensitive an ordinal scale would be preferable.

  3. What is an event listener? Why is it useful?

    4. An event listener is a listener than returns a callback when an event happens. As such you can set a listener on a paragraph to listen for a click or a mouse over. This in turn can then be used to tricker an event. It is very usefull because it provides a simple yet powerfull way for a user to interact with a graph or a website, that does not require the extensive user interfaces and buttons.

  4. What is your favorite D3 transition?

    5. The backInOut and BounceInOut are great! There is no doubt that they will be tried out first whenever a transition needs to be implemented, but as Scott says "easeCubicInOut is the default for a reason". See all here -> https://bl.ocks.org/mbostock/248bac3b8e354a9103c4

  5. Under what circumstances do you need to update your scale while doing transtions?

    6. A scenario where one would need to update the scale during a transition would be when a graph is updated to reflect changes in the data that it is visualizing. In more general words, a transition should update scales whenever new min or max values, or new categories are added to the data.

  6. Under what circumstances do you need to update your axes while doing transtions?

    7. When ever the scale is being updated to reflect new data The axes will also need be updated to reflect the changes in the scales.

  7. What's the point of on()?

    8. It is to give the ability to change the visualisation on the start or the end of a transition. However one should be careful not to initiate new transitions in the start() and end() because they will cancel out the started transition.

  8. Can you think of a real-world situation where you might want to add/remove bars to/from your histogram?

    9. For instance if one was to measure hours slept, the scale would need to be udated adding new days to reflect more data points as more days pass.

  9. Why might you want to use a key function when adding/removing data?

    10. Key function can be used when one wants more control over the process of adding or removing data. This is especilly helpful for adding or removing data that is not at the start or end of a visualization.
